Introduction
The Arm AI Optimization Challenge 2026 (Track 3: Mobile AI) demands an Android app
capable of running a multi-agent micro-swarm of sub-1B parameter AI models entirely on-
device to write, execute, validate, and repair code offline on Arm-powered phones. This report
synthesizes cutting-edge research and development as of mid-2026 to provide a detailed,
technical, and actionable roadmap for building PocketIDE. It covers model selection,
quantization, runtime frameworks (ExecuTorch, LiteRT-LM), Arm CPU optimizations, battery
efficiency, sandboxed code execution, multi-agent architecture, code editor integration, and
hackathon strategy tailored to Arm’s judging criteria.

Feasibility of Mixing Model Architectures
Mixing architectures (e.g., Qwen, Gemma, Llama) with different tokenizers is feasible but
introduces technical debt:
Tokenizer Conflicts: Different tokenizers may require additional normalization or
conversion layers
Memory Overhead: Loading multiple tokenizers increases memory usage and may
require careful memory management
Workarounds: Use ExecuTorch’s model loading/unloading API to maintain a flat memory
ceiling and avoid conflicts

Export Command and Process

INT4 Groupwise Quantization: Achieves up to 4x reduction in model size with minimal
accuracy loss (~3–5% drop in HumanEval pass@1)
Known Export Issues: Some models may have unsupported ops or tokenizer conflicts;
check GitHub issues for latest updates

Impact of Cache Locality

Battery Efficiency and Thermal Management (2025–2026)
Energy Efficiency Techniques
Model Quantization: INT4 quantization reduces energy consumption by up to 4x vs
FP32
Adaptive Core Selection: Android Dynamic Performance Framework (ADPF) enables
adaptive core usage without root access
MNN-AECS: Adaptive Energy-Centric Core Selection reduces energy consumption by
23–78% depending on configuration
Thermal Throttling Impact
Thermal throttling starts at 40°C, significantly degrading performance by 47°C
Android thermal management APIs (PowerManager, ThermalService) allow monitoring
and mitigation
Energy Consumption Measurement
Use Android APIs or hardware tools (Perfetto, Snapdragon Profiler) for fine-grained
energy profiling
TOPS per Watt metric indicates energy efficiency; INT4 quantization improves TOPS by
enabling more operations per clock cycle

Fine-Tuning Workflow for Sub-1B Models (2026)
LoRA Fine-Tuning Workflow
LoRA Rank: Start with r=16; target all linear layers
Unsloth Framework: Recommended for faster training and memory efficiency
Merging Adapters: Use llamafactory-cli export merge_config.yaml to merge
LoRA adapters into base model
Quantization: Apply INT4 groupwise quantization post-fine-tuning; benchmark
HumanEval before/after
Benchmarking and Quality Assurance
Use OpenCodeInstruct, Carllm, and SequenceR datasets for training and evaluation
Monitor training/validation loss to detect overfitting

Android Project Setup for JNI + ExecuTorch (2026)
Android Studio Setup
Install NDK r28+, configure CMakeLists.txt with Android toolchain
Add ExecuTorch AAR dependency via Maven Central or build from source
Configure Gradle to link native library
CMakeLists.txt Example
cmake_minimum_required(VERSION3.10)
project(PocketIDE)
set(ANDROID_ABI arm64-v8a)
set(CMAKE_TOOLCHAIN_FILE $ANDROID_NDK/build/cmake/android.toolchain.cmake)
add_library(executorchSTATICIMPORTED)
set_target_properties(executorchPROPERTIESIMPORTED_LOCATION${EXECUTORCH_LIB_PATH})
add_library(pocketideSHARED src/main/cpp/native-lib.cpp)
target_link_libraries(pocketide executorch)
Packaging .pte Model Files
Place .pte files in assets/ or raw/ directory
Access via Android AssetManager or native code
