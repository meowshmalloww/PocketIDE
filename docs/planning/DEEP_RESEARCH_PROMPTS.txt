PROMPT 1: ExecuTorch + Qwen2.5 Export Feasibility
================================================================================

I'm building an Android app that runs Qwen2.5-Coder-0.5B-Instruct entirely
on-device using ExecuTorch with KleidiAI + XNNPACK for Arm CPU optimization.

Research the following and provide detailed technical answers:

1. Can Qwen2.5-Coder-0.5B-Instruct be exported to ExecuTorch .pte format?
   What are the known issues (e.g., GitHub issue #9353 about SpinQuant export
   failing with assertion error in qc4w conversion)?

2. What is the exact export command for Qwen2.5 models? The architecture is
   Llama-compatible (RoPE, SwiGLU, RMSNorm, GQA). Does the --model "qwen2_5"
   flag work in examples.models.llama.export_llama?

3. What quantization modes work with Qwen2.5 on ExecuTorch? Does
   --preq_mode 8da4w_output_8da8w work, or are there alternatives?

4. Does HuggingFace optimum-executorch support Qwen2.5-0.5B? What recipes
   are available (XNNPACK fp32, quantized)?

5. Can ExecuTorch load LoRA adapters at runtime, or must LoRA weights be
   merged into the base model before export? This is critical — I want to
   use one base model with 3 different LoRA adapters (swapped at runtime)
   to save memory.

6. What is the memory footprint of Qwen2.5-Coder-0.5B in INT4 on Android?
   What decode tokens/sec can I expect on a mid-range Arm CPU (Cortex-A76/A78)?

7. Are there pre-built .pte files for Qwen2.5 models on Hugging Face that
   I can download directly?

PROMPT 2: Fine-Tuning Strategy for Multi-Agent Code Generation
================================================================================

I'm building a multi-agent system on Android with 3 specialized AI agents,
all running on-device:

- Agent 1 (Architect): Takes user prompt → outputs structured coding plan
- Agent 2 (Coder): Takes structured plan → generates code
- Agent 3 (Validator): Takes error trace → outputs repair instructions

Base model: Qwen2.5-Coder-0.5B-Instruct (0.49B params, Apache 2.0)

Research and provide detailed answers:

1. What datasets should I use to LoRA fine-tune each agent?
   - For Architect: task decomposition / planning datasets
   - For Coder: code generation from specs (HumanEval, MBPP, CodeAlpaca?)
   - For Validator: error analysis + code repair datasets

2. How do I generate training data via agent distillation? I want to use
   Qwen2.5-Coder-32B as the teacher model. What's the exact workflow?
   - How to generate teacher trajectories
   - How to format for LoRA SFT training
   - How many examples do I need (100? 1000? 10000?)

3. What LoRA configuration is optimal for a 0.5B model?
   - Rank (r): 8, 16, 32, 64?
   - Target modules: q_proj+v_proj only, or all linear layers?
   - Learning rate, batch size, epochs?

4. After LoRA fine-tuning, how do I merge adapters into the base model
   for ExecuTorch export? Can I export a merged model to .pte?

5. What quantization should I apply AFTER fine-tuning?
   - INT4 groupwise (group_size=32)?
   - SpinQuant?
   - QAT (quantization-aware training)?
   - How much accuracy loss vs FP32?

6. How do I benchmark code quality (HumanEval/MBPP pass@1) before and
   after fine-tuning + quantization to show measurable improvement?

PROMPT 4: Arm Optimization Deep Dive
================================================================================

I'm submitting to the Arm AI Optimization Challenge 2026 (Track 3: Mobile AI).
I need to show measurable Arm-specific optimizations for on-device LLM inference.

Research and provide detailed technical answers:

1. What is the exact performance difference between:
   - Standard XNNPACK (no KleidiAI) vs XNNPACK with KleidiAI
   - KleidiAI without SME2 vs with SME2
   - INT8 vs INT4 quantization on Arm CPU
   Provide specific tokens/sec numbers if available.

2. How do I build ExecuTorch with KleidiAI enabled for Android?
   What CMake flags are required?
   (-DEXECUTORCH_XNNPACK_ENABLE_KLEIDI=ON — what else?)

3. How do I detect at runtime whether the device supports:
   - i8mm instructions
   - SME2 instructions
   - NEON SIMD
   And fall back gracefully if not supported?

4. What is the Arm LLM Runner benchmark tool and how do I integrate it
   into my Android app for standardized benchmarking?

5. Can I use Arm Performix on mobile/Android devices, or is it cloud-only?
   If cloud-only, what Android-specific profiling tools should I use instead?

6. What is the theoretical maximum tokens/sec for a 0.5B INT4 model on:
   - Cortex-A76 (mid-range, 2.8GHz, no SME2)
   - Cortex-A78 (mid-range+, 2.4GHz, no SME2)
   - Cortex-X3 (flagship, 3.2GHz, with SME2)
